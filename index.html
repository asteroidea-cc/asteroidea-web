About

This portfolio serves as both a personal archive and a way to share practical tools developed for testing and validating technology—primarily within connected safety and care systems. The projects are built in response to real-world needs, such as simulating communication flows, isolating components for testing, or evaluating system behavior under controlled conditions.

The work is rooted in hands-on experimentation, integration, and iterative problem-solving. More projects will be added over time as new needs arise or when time allows.

If you're interested in the source code or want to collaborate, feel free to get in touch. This site is my way of contributing to the community and documenting solutions to common technical challenges.

SIA Premises Equipment (PE) Tester

A quick and simple tool for simulating and validating SIA-DCS alarm messages over TCP/IP. Designed for testing towards cloud-based alarm receivers, the solution provides control over SIA message structure and transmission.

Developed for the Arduino R4 WiFi platform
No Encryption Support
Key Features
  • Establishes Wi-Fi connectivity and resolves DNS for target alarm servers
  • Supports manual configuration of:
    • Sequence number
    • Receiver ID
    • Location prefix
    • Account identifier
  • Constructs compliant SIA-DCS messages with:
    • Framing using 0x0A and 0x0D
    • Hexadecimal payload length
    • CRC-16 checksum via compact algorithm
  • Sends structured messages over TCP and handles ACK/NAK responses
  • Offers real-time feedback through the Serial Monitor
Input Methods
  • Direct entry of two-character SIA codes (e.g., BA for Burglary)
  • Classification-based entry using classification,alarmcode format, mapped to over 200 predefined alarm types

BLE Scanner with Vendor Specific Frame Parsing

A versatile BLE scanning tool developed for the Arduino R4 WiFi platform, capable of interpreting Eddystone UID and vendor-specific advertisement frames. It enables real-time proximity estimation based on RSSI values and delivers structured, readable logging for development and diagnostics.

Developed for the Arduino R4 WiFi platform

Key Features
  • Continuous BLE scanning using the ArduinoBLE library
  • Detection and parsing of Eddystone UID frames and Minew-specific Info frames
  • Extraction of Namespace ID and Instance ID from Eddystone frames
  • Extraction of MAC address and device name from Minew frames
  • Consistent signal strength logging using RSSI values
  • Distance estimation based on RSSI and TX Power, calibrated per beacon type
  • Formatted and readable log output for easier diagnostics
  • Extended support for developer-specific frames, including Minew Info structures
  • Improved distance estimation logic with calibrated TX power for more accurate proximity readings
  • Consistent and unified logging format across different beacon types
  • Optimized Serial output for clearer diagnostics and easier debugging
Input Methods
  • Hardcoded MAC address filtering for targeted beacon recognition
  • Predefined byte arrays for matching specific Instance IDs
  • Automatic parsing of Eddystone UID and Minew formats without user input
